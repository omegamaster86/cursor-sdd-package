# 品質ゲート無効化ポリシー（共通）

## 目的
- 要件・設計・実装の変更が入った際に、どの品質ゲートを `not_run` / `stale` に戻すかを統一する。

## 基本原則
- **上流を変えたら下流を無効化** する。
- 実行実績がなく再実行が必要な状態は `not_run`、以前は通過していたが変更で古くなった状態は `stale` を使う。
- `traceability.status` は、要件/設計/実装証跡に影響したら少なくとも `stale` に戻す。

## 推奨マトリクス
| 変更イベント | review | final_check | traceability |
| --- | --- | --- | --- |
| 要件の生成・インポート・再生成 | not_run | not_run | stale |
| 設計の新規生成・大幅更新 | not_run | not_run | stale |
| 実装進行（要件実装、証跡追加） | stale | stale | stale |
| トレース更新（未マップなし） | (変更しない) | stale | current |
| トレース更新（未マップあり） | (変更しない) | stale | incomplete |
| レビュー実行 | passed/failed | (変更しない) | (変更しない) |
| 最終チェック実行 | (変更しない) | passed/failed | (変更しない) |

## 実装ルール
- コマンド本文では、上表に沿った無効化のみ記載する。
- 細かな条件分岐（例: `not_run` と `stale` の選択）は、差分の有無と既存状態で判断する。
- 迷った場合は安全側として `stale` を選び、再実行を促す。
